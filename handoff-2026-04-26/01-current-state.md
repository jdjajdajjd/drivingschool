# Current state

## Продукт

DriveDesk - SaaS для автошкол:

- публичная страница автошколы;
- онлайн-запись ученика;
- профиль ученика;
- админка автошколы;
- базовая superadmin-зона;
- каталог модулей и простая модель монетизации.

Основной сценарий сейчас: ученик открывает `/school/virazh`, видит профиль/кабинет, нажимает `Записаться на занятие`, проходит выбор филиала, инструктора, даты, времени, вводит данные и попадает на подтверждение.

## Стек

- Vite
- React
- TypeScript
- TailwindCSS
- React Router
- date-fns
- lucide-react
- framer-motion
- localStorage service layer

Команды:

```bash
npm run dev
npm run typecheck
npm run build
```

## Где сейчас главные ограничения

Проект все еще frontend-only MVP:

- нет backend API;
- нет настоящей БД;
- нет настоящей авторизации;
- нет runtime-ролей;
- данные живут в `localStorage`;
- seed/demo data остаются частью системы;
- multi-slot booking создает несколько отдельных записей;
- confirmation page фактически ориентирован на одну запись;
- нет настоящих SMS/Telegram/email уведомлений;
- нет полноценного E2E набора тестов.

## Деплой

Есть проблема с двумя путями деплоя:

- Cloudflare Pages project `drivingschool` с Git connection;
- Cloudflare Pages project `drivedesk`, куда деплоит GitHub Actions workflow.

Из-за этого пуш может визуально выглядеть как деплой в два проекта. Перед запуском нужно оставить один production path.
