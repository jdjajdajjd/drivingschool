# PROJECT BRIEF — DriveDesk

## Что это

DriveDesk — SaaS для автошкол с упором на онлайн-запись на вождение и операционную админку.

Текущий этап проекта:

- frontend-only MVP
- данные в `localStorage`
- без backend
- без Supabase
- без auth

## Основные роли

- ученик — самостоятельно выбирает слот и записывается
- администратор автошколы — управляет записями, слотами, инструкторами, филиалами и учениками
- инструктор — открывает личную ссылку и видит свои занятия

## Ключевые сценарии

### Публичная запись

- `/school/:schoolSlug`
- пошаговый flow: филиал → инструктор → дата → слот → контакты → подтверждение
- защита от записи в занятый слот
- лимит будущих активных записей на ученика

### Подтверждение записи

- `/booking/:bookingId`
- статус записи
- детали занятия
- `.ics`
- отмена записи

### Админка

- `/admin` — обзор и диагностика
- `/admin/bookings` — записи, фильтры, перенос, отмена, проведение
- `/admin/slots` — создание и контроль слотов
- `/admin/students` — список учеников
- `/admin/students/:studentId` — история и действия по ученику
- `/admin/instructors` — управление инструкторами и ссылками
- `/admin/branches` — управление филиалами

## Ограничения текущего этапа

- localStorage вместо сервера
- нет многопользовательской синхронизации в реальном времени между вкладками
- нет авторизации и ролей
- нет оплаты
- нет SMS/Telegram интеграций на runtime

## Что уже важно для качества

- строгий TypeScript
- бизнес-логика вынесена в сервисы
- мягкий SaaS UI без визуального шума
- mobile-friendly админка и booking flow
- устойчивость к битым ссылкам через integrity checks и safe fallbacks
