# DriveDesk

MVP SaaS для автошкол на `Vite + React + TypeScript + Tailwind + React Router + localStorage`.

Сейчас это полностью фронтендовый демо-продукт без backend, Supabase и auth. Основной сценарий уже рабочий: ученик может записаться на занятие, администратор видит и управляет данными в панели, инструктор открывает свою личную ссылку.

## Запуск

```bash
npm install
npm run dev
```

Открыть: `http://localhost:5173`

## Основные роуты

### Публичные

- `/` — демо-лендинг
- `/school/virazh` — публичная запись в автошколу
- `/booking/:bookingId` — подтверждение записи и отмена
- `/instructor/:token` — личная страница инструктора

### Админка

- `/admin` — обзор и операционная сводка
- `/admin/bookings` — записи: фильтры, отмена, проведение, перенос
- `/admin/slots` — слоты: список, одиночное и массовое создание, статусы
- `/admin/students` — список учеников
- `/admin/students/:studentId` — профиль и история ученика
- `/admin/instructors` — управление инструкторами и личными ссылками
- `/admin/branches` — управление филиалами
- `/admin/modules` — каталог модулей
- `/admin/modules/:moduleId` — карточка модуля
- `/superadmin` — демо-панель платформы

## Что уже работает

### Booking flow

- выбор филиала
- выбор инструктора
- выбор даты
- выбор слота
- ввод имени и телефона
- нормализация телефона к виду `7xxxxxxxxxx`
- поиск или создание `student` по `normalizedPhone`
- создание `booking`
- перевод `slot.status` в `booked`
- редирект на `/booking/:bookingId`
- скачивание `.ics`
- отмена записи с освобождением слота
- лимит будущих активных записей на ученика

### Админка

- dashboard с метриками, ближайшими занятиями и простыми диагностическими сигналами
- bookings со статусами, поиском, фильтрами, отменой, отметкой “проведено” и переносом
- slots со списком, статусами, удалением свободных слотов, отменой/восстановлением и массовым созданием
- students со списком, лимитами и историей записей
- instructors с созданием, редактированием, включением/выключением и личной ссылкой
- branches с созданием, редактированием и безопасным удалением

## Demo data

При первом запуске автоматически создаются демо-данные для школы `virazh`:

- 1 школа
- 3 филиала
- несколько инструкторов
- слоты на ближайшие дни
- demo bookings
- demo students

Seed создаётся только если данных ещё нет. Для новой версии схемы seed переинициализируется через `SEED_VERSION`.

## Как протестировать demo flow

### Сценарий ученика

1. Открыть `/school/virazh`
2. Выбрать филиал, инструктора, дату и слот
3. Ввести имя и телефон
4. Создать запись
5. Открыть `/booking/:bookingId`
6. Скачать `.ics`
7. Отменить запись и убедиться, что слот снова свободен

### Сценарий админа

1. Открыть `/admin/bookings`
2. Найти созданную запись
3. Отменить, отметить проведённой или перенести
4. Проверить `/admin/students`
5. Проверить `/instructor/:token`

## Как сбросить demo data

В проекте есть `resetDemoData()` в `src/services/seed.ts`. Для ручного сброса в браузере можно очистить `localStorage` и перезагрузить страницу:

```js
localStorage.clear()
location.reload()
```

## Текущее ограничение архитектуры

- все данные живут в `localStorage`
- нет backend API
- нет auth
- нет Supabase
- нет live-sync между вкладками

Это осознанный MVP-этап, чтобы быстро проверить продуктовый сценарий и UI админки.
